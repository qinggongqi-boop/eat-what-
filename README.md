# 吃罢了？

一个低成本静态 H5 上班带饭搭配器。

## 使用

直接打开 `index.html` 即可运行，也可以部署到 Cloudflare Pages、Vercel、GitHub Pages 等静态托管平台。

## 已实现

- 随机生成蛋白/蔬菜组合，数量可在设置中调整。
- 自选搭配菜谱，并根据当前组合推荐补充菜品。
- 喜欢收藏、不喜欢屏蔽、多推收藏菜。
- 菜谱列表、标签筛选、详情查看。
- 自定义菜谱本地保存，支持 JSON 导入/导出。

## 数据说明

用户设置、收藏、不喜欢、自定义菜谱都保存在浏览器 `localStorage` 中，无需后端。

## 菜谱内容说明

当前内置菜谱是用于产品原型的种子内容，由 Codex 按常见家常菜做法整理生成，并未从第三方菜谱网站抓取或复制。它们适合验证页面流程、卡片信息结构和带饭搭配逻辑，但不应直接视为正式上线内容。

正式发布前建议逐道人工复核：

- 食材和调料是否完整、常见、容易购买。
- 步骤是否足够简洁，厨房新手是否能照做。
- 火候、加热、隔夜保存和微波复热是否安全。
- 热量估算是否只作为“大概参考”，避免承诺精准营养数据。

## 社交化路线

### 前期：无后端快照分享

- 保持静态 H5，不增加登录和服务器。
- 使用“生成共享链接”分享随机结果、自选组合、收藏、不喜欢、设置和自定义菜谱。
- 适合情侣、室友、朋友之间临时讨论：“明天带这组行不行？”
- 缺点是不同设备不会实时同步，需要重新生成链接。

### 中期：饭搭子房间

- 增加轻后端和房间码，例如一个 6 位数字/字母房间。
- 2-3 个设备共享同一个房间里的收藏、不喜欢、随机设置、周计划和购物清单。
- 不强制手机号登录，先用房间码 + 本地昵称降低使用门槛。
- 支持“我想吃/不想吃”投票，方便两个人共同决定一周带饭。

### 后期：账号、小程序与关系链

- 如果验证出稳定需求，再做微信小程序或账号体系。
- 支持跨设备长期保存、家庭/情侣/室友多个饭搭子空间。
- 增加评论、跟做反馈、共享菜谱包和公开精选带饭计划。
- 这阶段才考虑审核、内容运营、云数据库、权限和数据迁移。
